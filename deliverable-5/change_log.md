## Artifact Progression

The Deliverable 5 package builds on the earlier Benji assignment work:

1. Requirements established the payroll problem, stakeholders, and core modules.
2. Specification and design work defined the screens, use cases, messages, and architecture.
3. Project planning identified resources, schedule, risks, and QA responsibilities.
4. Component-level design mapped functional, middleware, and UI components.
5. Deliverable 5 connects those pieces into a user journey map, working prototype, and functional testing evidence.

## Current Submission Scope

The repository intentionally contains only the Deliverable 5 package needed for grading. Larger implementation work is outside this clean submission scope.
